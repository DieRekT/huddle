# Huddle Demo Video Script

This script outlines key moments and features to showcase in a Huddle demo video.

## Video Structure

**Total Length**: 3-5 minutes
**Target Audience**: Deaf/HoH users, accessibility advocates, developers

## Scene 1: Introduction (0:00 - 0:30)

### Visual
- Huddle logo and tagline: "Read the room"
- Brief text overlay: "Real-time transcription for deaf and hard-of-hearing users"

### Narration (Optional)
"Welcome to Huddle, a real-time transcription and AI-powered summarization tool designed specifically for deaf and hard-of-hearing users."

### Key Points
- Problem statement: Missing conversation context
- Solution: Huddle provides real-time captions and summaries
- Multi-location support

## Scene 2: Room Creation (0:30 - 1:00)

### Visual
- Open Huddle in browser
- Enter name: "Sarah"
- Select "Viewer" role
- Show optional passcode field
- Click "Create Room"
- Room code appears: "A1B2C3"
- Click "Invite" button
- QR code appears

### Narration
"Creating a room is simple. Enter your name, optionally add a passcode for security, and you'll get a unique room code. Share the QR code or link with participants."

### Key Points
- Easy room creation
- Optional passcode protection
- QR code sharing
- Room code display

## Scene 3: Mic Setup (1:00 - 1:30)

### Visual
- Open mic link on phone
- Enter name: "John"
- Enter room passcode (if set)
- Click "Join Room"
- Show consent checkbox
- Check consent
- Click "Start Mic"
- Show "Mic LIVE" indicator pulsing
- Show mic statistics

### Narration
"Participants join as mics. After consenting, they can start streaming audio. The viewer sees real-time status updates."

### Key Points
- Easy joining process
- Consent requirement
- Visual feedback (Mic LIVE indicator)
- Multi-device support

## Scene 4: Real-Time Transcription (1:30 - 2:00)

### Visual
- Viewer screen showing live transcript
- Multiple speakers talking
- Transcript updating in real-time
- Show speaker labels
- Show mic health chips (LIVE/QUIET)
- Topic and summary updating

### Narration
"As people speak, Huddle transcribes everything in real-time. Each speaker is clearly labeled, and the AI continuously updates the topic and summary."

### Key Points
- Real-time transcription
- Speaker identification
- Live topic updates
- Mic health monitoring

## Scene 5: Full Room Overview (2:00 - 2:45)

### Visual
- Click "Read the Room" button
- Panel opens showing:
  - Overview text
  - Key points list
  - Decisions section
  - Next steps section
- Show "Copy" button
- Click copy button
- Show toast: "Full Room Overview copied to clipboard"
- If long meeting, show segment count note

### Narration
"The Full Room Overview provides a comprehensive summary of the entire conversation, including key points, decisions, and next steps. You can copy it with one click."

### Key Points
- Comprehensive summary
- Decisions and next steps
- Copy functionality
- Chunked summarization for long meetings

## Scene 6: Accessibility Features (2:45 - 3:15)

### Visual
- Click menu (⋯ button)
- Show accessibility section
- Toggle font size: Normal → Large → Extra Large
- Toggle high contrast mode: Off → On
- Show visual changes
- Demonstrate keyboard navigation (Tab through interface)

### Narration
"Huddle includes built-in accessibility features. Adjust font size, enable high contrast mode, and navigate entirely with your keyboard."

### Key Points
- Font size options
- High contrast mode
- Keyboard navigation
- Accessibility-first design

## Scene 7: Catch-Up & Export (3:15 - 3:45)

### Visual
- Click "Catch-up" button
- Show catch-up panel with recent summary
- Click "Export" button
- Show export options (Markdown/JSON)
- Download file
- Show "Save & Clear" option

### Narration
"Missed something? Use Catch-up for a quick summary. Export your session data, or use Save & Clear to copy a summary and start fresh."

### Key Points
- Catch-up feature
- Export functionality
- Save & Clear workflow

## Scene 8: Multi-Location Demo (3:45 - 4:15)

### Visual
- Show multiple devices (laptop, phone, tablet)
- All connected to same room
- Different speakers on each device
- Unified transcript on viewer screen
- Show mic roster with all devices

### Narration
"Huddle excels in multi-location scenarios. Multiple people in different places can all contribute, while the viewer sees one unified transcript."

### Key Points
- Multi-location support
- Unified transcript
- Device flexibility

## Scene 9: Closing (4:15 - 4:30)

### Visual
- Huddle logo
- Website/GitHub link
- "Try Huddle today" call-to-action

### Narration
"Huddle makes conversations accessible. Try it today and experience the difference."

### Key Points
- Call to action
- Where to find more information
- Open source nature

## Production Notes

### Screen Recording Tips
- Use high resolution (1080p minimum)
- Show cursor movements clearly
- Highlight important UI elements
- Use zoom for mobile demonstrations
- Include timestamps in corner

### Accessibility
- Provide captions/subtitles
- Use clear, simple language
- Show keyboard shortcuts on screen
- Include audio description option
- Provide transcript of narration

### Editing
- Smooth transitions between scenes
- Add text overlays for key features
- Include brief pauses for processing
- Highlight important moments
- Keep pace engaging but not rushed

### Distribution
- YouTube with captions
- GitHub repository
- Documentation site
- Social media snippets (30-60 second cuts)

## Key Moments to Highlight

1. **0:45** - Room creation with passcode
2. **1:15** - Consent checkbox (privacy focus)
3. **2:00** - Full Room Overview with decisions/next_steps
4. **2:45** - Accessibility features demonstration
5. **3:30** - Multi-device, multi-location setup

Focus on:
- Room creation (10s)
- Real-time transcription (20s)
- Full Room Overview (15s)
- Accessibility toggle (10s)
- Closing (5s)
